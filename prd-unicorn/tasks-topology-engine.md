## Sprint 1: Topology Integration (2 weeks)

### TASK-TOPO-001: Relationship Tabs on Resource Detail Pages (P0) ✅
**Goal:** Surface topology data outside the topology page — the single most impactful UX improvement
**Acceptance Criteria:**
- [x] Add "Relationships" tab to Deployment, Pod, Service, ConfigMap, Secret, PVC detail pages
- [x] Show mini React Flow graph with 1-hop neighbors (incoming + outgoing relationships)
- [x] Each node clickable — navigates to that resource's detail page
- [x] Lazy-load: fetch topology data only when tab is selected
- [x] Show relationship type labels on edges (owns, selects, mounts, routes-to)
- [x] Graceful empty state: "No relationships detected" with link to topology page

### TASK-TOPO-002: URL-Encoded Topology State (P0) ✅
**Goal:** Enable sharing topology views via URL
**Acceptance Criteria:**
- [x] Encode in URL: view mode, namespace filter, zoom level, selected node
- [x] Format: `?view=workload&ns=production&zoom=0.75&selected=deploy/nginx`
- [x] Browser back/forward respects topology state changes
- [x] Deep-linking from other pages preserves state

### TASK-TOPO-003: Topology Performance — Progressive Disclosure (P1) ✅
**Goal:** Handle clusters with 500+ resources without hitting 250-node cap
**Acceptance Criteria:**
- [x] Default view: show namespace groups as collapsed super-nodes with resource counts
- [x] Click namespace to expand and show contained resources
- [x] Expanded namespaces lazy-load their resources
- [x] Update MAX_VISIBLE_NODES to 500 for expanded view
- [x] Virtual rendering: only render nodes in viewport

## Sprint 3: Advanced Topology (3 weeks)

### TASK-TOPO-006: CRD Relationship Detection (P1) ✅
**Goal:** Include custom resources in topology when they reference core resources
**Acceptance Criteria:**
- [x] New `CRDRelationshipMatcher` that introspects CRD schemas
- [x] Detect field references to core resource kinds (e.g., VirtualService → Service)
- [x] Detect ownerReferences from CRD instances to core resources
- [x] Show CRD nodes with generic "Custom" category color
- [x] User opt-in: toggle "Show Custom Resources" in topology toolbar

### TASK-TOPO-007: Gateway API in Topology (P1) ✅
**Goal:** Visualize modern Kubernetes networking
**Acceptance Criteria:**
- [x] Add Gateway, GatewayClass, HTTPRoute, GRPCRoute to topology
- [x] New `GatewayMatcher`: Gateway → GatewayClass, HTTPRoute → Service, HTTPRoute → Gateway
- [x] Gateway nodes in Networking category (cyan)
- [x] HTTPRoute edges show path matching rules

### TASK-TOPO-008: Historical Topology Comparison (P2) ✅
**Goal:** Understand "what changed" for post-incident analysis
**Acceptance Criteria:**
- [x] Store topology snapshots (hourly, on-demand, on significant change)
- [x] "Compare" mode: select two timestamps
- [x] Diff visualization: green=added nodes/edges, red=removed, amber=changed
- [x] Summary: "5 nodes added, 2 removed, 8 edges changed"
- [x] Export diff as report

### TASK-TOPO-009: Cross-Cluster Topology (P2) ✅
**Goal:** Fleet-level visualization for platform engineers
**Acceptance Criteria:**
- [x] "Fleet" view mode showing clusters as top-level nodes
- [x] Click cluster to expand into namespace-level view
- [x] Cross-cluster edges for: shared storage, DNS routing, service mesh
- [x] Aggregate health status per cluster
